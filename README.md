# Personal Network

这是一个用于记录、共享和探讨个人网络架构方案的开源仓库。

项目重点关注日常可落地的网络实践，例如寝室自建局域网、NAS 异地访问、VPN 组网、远程维护、安全边界设计等。这里的内容更偏向方案思路、拓扑设计、配置原则和踩坑记录，而不是直接复制到生产环境的完整私有配置。

## 目标

- 整理个人网络架构的设计思路和演进过程
- 记录寝室、家庭、小型工作室等场景下的组网方案
- 分享 NAS 异地访问、VPN、内网穿透、远程备份等实践经验
- 讨论安全、可维护性、成本、性能之间的取舍
- 为相似场景的人提供可参考的公开方案

## 主要方向

### 寝室自建局域网

面向宿舍、合租、公寓等网络环境，记录如何在受限条件下构建稳定、清晰、易维护的个人局域网。

可能包含：

- 路由器、交换机、无线 AP 的角色划分
- 主路由、旁路由、软路由方案对比
- VLAN、子网隔离、访客网络设计
- 多设备接入、地址规划和 DHCP 管理
- 校园网、认证网关、NAT 环境下的网络设计思路

### NAS 异地访问

面向个人 NAS、家庭服务器、实验室设备等场景，记录如何在外部网络中安全访问内部服务。

可能包含：

- VPN 组网方案
- 远程访问 NAS 管理后台和文件服务
- 移动设备、笔记本、远端服务器之间的互联
- 异地备份和数据同步设计
- 权限控制、访问审计和暴露面收敛

### VPN 与远程组网

记录不同远程组网方式的适用场景、优缺点和安全边界。

可能包含：

- WireGuard、OpenVPN、Tailscale、ZeroTier 等方案对比
- 点对点、星型、全互联组网模型
- 出口节点、跳板节点和中继节点设计
- 多地点、多设备访问策略
- 密钥轮换、访问控制和故障恢复

## 安全与隐私原则

本仓库是公开仓库，所有内容都应默认面向互联网公开。

请不要提交任何敏感信息，包括但不限于：

- 真实公网 IP、内网 IP、域名和端口映射
- token、密钥、密码、cookie、证书私钥
- 设备序列号、MAC 地址、账号、邮箱、手机号
- 能够定位到个人环境、学校、宿舍或家庭位置的细节

示例内容应使用占位符或文档保留地址，例如：

- `example.com`
- `192.0.2.10`
- `10.0.0.0/24`
- `<YOUR_TOKEN_HERE>`
- `<NAS_HOST>`
