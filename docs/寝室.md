# 树莓派寝室软路由 + NAS + Minecraft 服务器方案

## 1. 目标

本方案用于在寝室环境中，将学校有线网络通过树莓派转换为个人 WiFi 热点，同时让树莓派承担小型寝室服务器功能。

主要目标：

- 使用树莓派通过有线网口接入学校网络
- 使用树莓派 WiFi 创建个人热点
- 手机、电脑、平板等设备连接树莓派热点上网
- 在树莓派后方形成一个相对独立的小型局域网
- 树莓派作为寝室 NAS，通过 SMB 提供文件共享
- 树莓派作为 Minecraft 寝室服务器，供内网设备联机
- 尽量避免暴露校园网 IP、账号、MAC、宿舍位置等敏感信息

---

## 2. 网络结构

基础结构如下：

```text
学校有线网络 / 墙上网口
        ↓
树莓派 eth0
        ↓
树莓派 wlan0 热点
        ↓
手机 / 笔记本 / 平板 / 其他设备
```

树莓派角色：

```text
eth0：接入学校网络，作为 WAN 侧
wlan0：创建寝室 WiFi 热点，作为 LAN 侧
树莓派本机：网关 + NAS + Minecraft 服务器
```

逻辑结构：

```text
学校网络侧：
树莓派作为普通有线客户端接入学校网络

寝室内网侧：
树莓派作为网关，为个人设备分配内网 IP
树莓派同时提供 SMB 文件共享和 Minecraft 服务
```

示例内网规划：

```text
树莓派热点网关：192.168.50.1
个人设备网段：192.168.50.0/24
SMB 访问地址：192.168.50.1
Minecraft 服务器地址：192.168.50.1
```

说明：

- 学校网络侧只直接看到树莓派
- 个人设备位于树莓派后方的 NAT 内网中
- 内网设备之间可以互相访问
- 默认情况下，学校网络中的其他设备无法直接访问寝室内网设备
- SMB 和 Minecraft 服务优先只面向寝室内网开放

---

## 3. 推荐方案

推荐使用：

```text
有线接入 + WiFi 热点 + NAT 共享 + 内网服务
```

不推荐使用纯桥接模式。

原因：

- 校园网通常存在认证、MAC 绑定或设备数量限制
- WiFi 与有线网络之间直接桥接可能不稳定
- NAT 模式更容易配置，也更适合保护内网设备
- 学校网络侧通常只会识别树莓派这一个上网设备
- SMB 和 Minecraft 服务可以集中限制在寝室内网中

## 6. 基础网络配置

### 6.1 确认有线网络可用

将学校网线插入树莓派有线网口。

查看网卡状态：

```bash
ip a
```

确认有线网口通常为：

```text
eth0
```

测试网络：

```bash
ping -c 4 example.com
```

如果学校网络需要网页登录认证，可以先在树莓派浏览器中完成认证。

---

### 6.2 创建 WiFi 热点

使用 NetworkManager 创建热点：

```bash
sudo nmcli device wifi hotspot ifname wlan0 con-name dorm-hotspot ssid Pi-Dorm password "ChangeThisPassword"
```

说明：

```text
wlan0：树莓派无线网卡
dorm-hotspot：热点连接名称
Pi-Dorm：WiFi 名称，可自行修改
ChangeThisPassword：WiFi 密码，请修改为强密码
```

密码建议：

```text
不少于 12 位
包含大小写字母、数字和符号
不要使用生日、姓名、手机号等信息
```

---

### 6.3 固定热点内网网段

将热点网关固定为：

```text
192.168.50.1/24
```

执行：

```bash
sudo nmcli connection modify dorm-hotspot ipv4.addresses 192.168.50.1/24
sudo nmcli connection modify dorm-hotspot ipv4.method shared
sudo nmcli connection down dorm-hotspot
sudo nmcli connection up dorm-hotspot
```

连接热点的设备会获得类似地址：

```text
192.168.50.x
```

默认网关：

```text
192.168.50.1
```

## 7. NAS / SMB 文件共享规划

### 7.1 目标

树莓派作为寝室 NAS，提供局域网文件共享。

用途包括：

```text
照片备份
学习资料同步
临时文件中转
项目文件存放
寝室内设备共享文件
```

推荐只允许寝室内网访问 SMB，不建议开放到学校网络侧。

---

### 7.2 推荐目录结构

示例目录：

```text
/mnt/storage
├── public
├── private
├── photos
├── study
├── projects
└── minecraft-backup
```

说明：

```text
public：寝室内临时共享
private：个人私有文件
photos：照片与相册数据
study：学习资料
projects：项目文件
minecraft-backup：Minecraft 存档备份
```

文档中不记录真实硬盘序列号、真实挂载 UUID 或具体个人文件名。

---

### 7.3 安装 Samba

```bash
sudo apt update
sudo apt install samba
```

创建共享目录：

```bash
sudo mkdir -p /mnt/storage/public
sudo mkdir -p /mnt/storage/private
sudo mkdir -p /mnt/storage/photos
sudo mkdir -p /mnt/storage/study
sudo mkdir -p /mnt/storage/projects
sudo mkdir -p /mnt/storage/minecraft-backup
```

设置目录权限时，建议根据实际用户管理。不要为了省事长期使用全开放权限。

---

### 7.4 添加 SMB 用户

将当前用户加入 Samba：

```bash
sudo smbpasswd -a $USER
```

启用用户：

```bash
sudo smbpasswd -e $USER
```

设置 Samba 密码时，不要和学校账号、系统登录密码完全相同。

---

### 7.5 示例 Samba 配置

编辑配置文件：

```bash
sudo nano /etc/samba/smb.conf
```

示例配置：

```ini
[public]
   path = /mnt/storage/public
   browseable = yes
   read only = no
   guest ok = no
   valid users = %S

[private]
   path = /mnt/storage/private
   browseable = yes
   read only = no
   guest ok = no
   valid users = %S

[study]
   path = /mnt/storage/study
   browseable = yes
   read only = no
   guest ok = no
   valid users = %S

[photos]
   path = /mnt/storage/photos
   browseable = yes
   read only = no
   guest ok = no
   valid users = %S

[projects]
   path = /mnt/storage/projects
   browseable = yes
   read only = no
   guest ok = no
   valid users = %S

[minecraft-backup]
   path = /mnt/storage/minecraft-backup
   browseable = yes
   read only = no
   guest ok = no
   valid users = %S
```

测试配置：

```bash
testparm
```

重启 Samba：

```bash
sudo systemctl restart smbd
```

---


macOS：

```text

```

## 8. Minecraft 寝室服务器规划

### 8.1 目标

树莓派作为 Minecraft 寝室服务器，供连接寝室热点的设备联机使用。

推荐用途：

```text
寝室小规模联机
好友局域网服务器
轻量插件服
学习 Linux 服务管理
```

不建议用途：

```text
大型公开服务器
大量玩家在线
重型整合包服务器
高强度红石 / 生电服务器
```

树莓派 5 可以承担轻量 Minecraft 服务器，但建议控制在线人数、视距和插件数量。

---

### 8.2 网络访问方式

服务器地址：

```text
192.168.50.1
```

默认端口：

```text
25565
```

连接方式：

```text
Minecraft 客户端 → 多人游戏 → 添加服务器 → 192.168.50.1:25565
```

建议只允许寝室内网访问，不对学校网络侧开放。

---

### 8.3 安装 Java

具体 Java 版本取决于 Minecraft 服务端版本。

常见安装方式：

```bash
sudo apt update
sudo apt install openjdk-21-jre-headless
```

检查版本：

```bash
java -version
```

---

### 8.4 创建服务器目录

```bash
mkdir -p ~/minecraft-server
cd ~/minecraft-server
```

下载服务端文件后，将服务端 jar 文件放入该目录。

示例命名：

```text
server.jar
```

首次启动：

```bash
java -Xms1G -Xmx2G -jar server.jar nogui
```

首次启动后会生成：

```text
eula.txt
```

编辑：

```bash
nano eula.txt
```

将：

```text
eula=false
```

改为：

```text
eula=true
```

再次启动：

```bash
java -Xms1G -Xmx2G -jar server.jar nogui
```

---

### 8.5 推荐 server.properties 设置

编辑：

```bash
nano server.properties
```

推荐轻量配置：

```properties
server-port=25565
online-mode=true
max-players=4
view-distance=6
simulation-distance=4
enable-command-block=false
spawn-protection=8
white-list=true
motd=Dorm Minecraft Server
```

说明：

```text
max-players：建议 2–4 人
view-distance：建议 6 左右，降低性能压力
simulation-distance：建议 4 左右
white-list：建议开启白名单
online-mode：正版验证，建议开启
```

---

### 8.6 白名单管理

在服务器控制台中执行：

```text
whitelist on
whitelist add PlayerName
whitelist list
```

如果是 Bedrock 或特殊服务端，白名单方式可能不同。

---

### 8.7 使用 systemd 管理 Minecraft 服务

创建服务文件：

```bash
sudo nano /etc/systemd/system/minecraft.service
```

示例：

```ini
[Unit]
Description=Minecraft Dorm Server
After=network.target

[Service]
WorkingDirectory=/home/pi/minecraft-server
ExecStart=/usr/bin/java -Xms1G -Xmx2G -jar server.jar nogui
Restart=on-failure
RestartSec=10
User=pi

[Install]
WantedBy=multi-user.target
```

注意：

```text
WorkingDirectory 需要改成实际服务器目录
User 需要改成实际运行用户
```

重新加载 systemd：

```bash
sudo systemctl daemon-reload
```

启动服务：

```bash
sudo systemctl start minecraft
```

设置开机自启：

```bash
sudo systemctl enable minecraft
```

查看状态：

```bash
sudo systemctl status minecraft
```

查看日志：

```bash
journalctl -u minecraft -f
```

### 8.8 Minecraft 存档备份

建议定期备份：

```text
world
world_nether
world_the_end
server.properties
whitelist.json
ops.json
banned-players.json
banned-ips.json
```

示例备份命令：

```bash
mkdir -p /mnt/storage/minecraft-backup
tar -czf /mnt/storage/minecraft-backup/mc-backup-$(date +%Y%m%d-%H%M).tar.gz ~/minecraft-server
```

## 9. 防火墙建议

如果系统开启了 UFW，需要避免防火墙阻止热点共享、SMB 和 Minecraft 内网访问。

查看状态：

```bash
sudo ufw status verbose
```

允许寝室内网访问树莓派：

```bash
sudo ufw allow in on wlan0 from 192.168.50.0/24
```

允许 SSH 仅从寝室内网访问：

```bash
sudo ufw allow in on wlan0 from 192.168.50.0/24 to any port 22
```

允许 SMB 仅从寝室内网访问：

```bash
sudo ufw allow in on wlan0 from 192.168.50.0/24 to any port 445
sudo ufw allow in on wlan0 from 192.168.50.0/24 to any port 139
```

允许 Minecraft 仅从寝室内网访问：

```bash
sudo ufw allow in on wlan0 from 192.168.50.0/24 to any port 25565 proto tcp
```

不建议在学校网络侧开放这些端口：

```text
22/tcp
139/tcp
445/tcp
25565/tcp
Web 管理面板端口
```

如果需要远程访问，优先使用 VPN，而不是直接开放端口。

## 10. 服务暴露原则

推荐暴露范围：

```text
SMB：仅寝室内网
Minecraft：仅寝室内网
SSH：仅寝室内网或可信 VPN
Web 管理面板：仅寝室内网或可信 VPN
```

不推荐：

```text
SMB 暴露到校园网
Minecraft 暴露到整个校园网
SSH 允许整个校园网访问
使用弱密码
关闭白名单后让陌生设备进入 Minecraft 服务器
```

---

## 11. 性能与稳定性建议

### 11.1 树莓派负载

同时运行以下服务时：

```text
WiFi 热点
NAT 转发
SMB 文件共享
Minecraft 服务器
其他后台服务
```

建议：

```text
使用主动散热
尽量使用 SSD，而不是长期依赖 microSD
Minecraft 分配内存不要过高
限制 Minecraft 在线人数和视距
避免同时进行大量 SMB 文件传输和 Minecraft 服务器运行
```

---

### 11.2 存储建议

推荐：

```text
系统盘：microSD 或 SSD
数据盘：外接 SSD / 移动硬盘
重要数据：至少另有一份备份
```

不建议：

```text
所有重要文件只放一块移动硬盘
Minecraft 存档完全不备份
SMB 共享目录权限全开放
长时间高负载写入 microSD
```

---

### 11.3 Minecraft 参数建议

树莓派 5 轻量服务器建议：

```text
玩家数量：2–4 人
最大内存：2G 左右起步
视距：6
模拟距离：4
插件：尽量少
整合包：谨慎
```

示例启动参数：

```bash
java -Xms1G -Xmx2G -jar server.jar nogui
```

如果系统内存较大，并且没有运行太多其他服务，可以适当提高到：

```bash
java -Xms1G -Xmx3G -jar server.jar nogui
```

## 13. 故障排查

### 13.1 设备能连 WiFi，但不能上网

检查树莓派本身是否能上网：

```bash
ping -c 4 example.com
```

重启热点：

```bash
sudo nmcli connection down dorm-hotspot
sudo nmcli connection up dorm-hotspot
```

---

### 13.2 SMB 访问失败

检查 Samba 状态：

```bash
systemctl status smbd
```

检查配置是否正确：

```bash
testparm
```

检查防火墙：

```bash
sudo ufw status verbose
```

确认客户端连接的是：

```text
192.168.50.1
```

---

### 13.3 Minecraft 无法连接

检查服务状态：

```bash
systemctl status minecraft
```

检查端口监听：

```bash
ss -tulpn | grep 25565
```

检查防火墙：

```bash
sudo ufw status verbose
```

确认客户端连接地址：

```text
192.168.50.1:25565
```

确认客户端和服务器版本一致。

---

### 13.4 Minecraft 卡顿

可以尝试：

```text
降低 view-distance
降低 simulation-distance
减少玩家数量
减少插件数量
避免大型红石机器
避免同时进行大文件 SMB 传输
使用 SSD 存放服务器目录
增加主动散热
```

## 14. 敏感信息保护

文档中不要写入：

```text
真实校园网 IP
真实公网 IP
学校账号
统一认证账号
密码
MAC 地址
宿舍楼栋和房间号
个人真实姓名
硬盘序列号
真实设备名
外部访问地址
```

可以使用占位符：

```text
<校园网账号>
<热点名称>
<强密码>
<设备名>
<内网网段>
<服务端口>
```

推荐把含敏感信息的内容单独放在本地私密文件中，不上传 GitHub。

例如：

```text
secrets.local.md
.env
.env.local
```

并在 `.gitignore` 中加入：

```gitignore
secrets.local.md
.env
.env.local
```

## 16. 总结

本方案使用树莓派作为寝室小型软路由和个人服务器，实现：

```text
有线校园网接入
WiFi 热点共享
个人内网隔离
SMB 文件共享
Minecraft 寝室服务器
后续 VPN / DNS / NAS 扩展
```

推荐优先使用 NAT 共享模式，避免纯桥接。

SMB、Minecraft、SSH 等服务应优先限制在寝室内网或可信 VPN 中，不建议直接暴露到学校网络或公网。
